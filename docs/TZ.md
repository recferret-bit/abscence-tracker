# Техническое задание: Таблица учёта отпусков, праздников и больничных
**Продукт:** Таблица учёта отпусков, праздничных и больничных дней  
**Платформа:** Google Sheets  
**Версия документа:** 1.0  
**Дата:** 2026-04-08

## 1. Назначение
Создать единую Google Sheets-таблицу для учёта всех типов отсутствий сотрудников компании.
Все записи должны отображаться на одном листе с визуальным различием по категориям.

Текущий ориентир — таблица «График отпусков 2026 FinHarbor», где отсутствия сейчас распределены по 3 листам. В новой версии все категории должны быть объединены в один лист.

## 2. Область учёта
Система должна учитывать следующие категории дней:
- `VACATION` — отпуск
- `HOLIDAY` — праздничный день
- `SICK` — больничный

## 3. Функциональные требования
### REQ-01. Единый лист
- Все категории отсутствий должны вестись на одном листе.
- Пользователь должен видеть полную картину по сотрудникам без переключения между листами.

### REQ-02. Визуализация категорий
Для каждого типа дня должна использоваться отдельная цветовая индикация:
- Отпуск → зелёный
- Праздничный день → синий
- Больничный → красный

### REQ-03. Учёт отпускных дней
- Лимит: 20 дней в год.
- В расчёт включаются только рабочие дни (пн–пт).
- Неиспользованные дни переносятся на следующий год.
- Перенесённые дни могут быть использованы до 30 июня следующего года.

### REQ-04. Учёт праздничных дней
- Лимит: 17 дней в год.
- Даты задаются вручную.
- В расчёт включаются только рабочие дни (пн–пт).
- Неиспользованные праздничные дни сгорают в конце года и не переносятся.

### REQ-05. Учёт больничных дней
- Больничные дни учитываются без лимита.
- Требуется только факт учёта использованных дней.

### REQ-06. Автоматическое определение выходных
- Таблица автоматически определяет выходные дни.
- Выходные дни визуально выделяются.
- Выходные дни не включаются в расчёт ни для одного типа отсутствия.

### REQ-07. Расчёт показателей
Для каждого сотрудника отображаются:
- Рабочий год
- Сегодняшняя дата
- Дата начала работы
- Начислено на текущую дату
- Начислено на год
- Использовал
- Остаток
- Руководитель
- Отдел

Остатки отображаются отдельно для отпускных и праздничных дней: на текущую дату и на календарный год.

### REQ-08. Структура столбцов
Таблица должна содержать следующие столбцы:
- Сотрудник
- Тип дня
- Рабочий год
- Сегодняшняя дата
- Дата начала работы
- Начислено на тек. дату
- Начислено на год
- Использовал
- Остаток
- Руководитель
- Отдел

### REQ-09. Валидация столбца «Тип дня»
- Столбец «Тип дня» использует выпадающий список.
- Допустимые значения: `VACATION`, `HOLIDAY`, `SICK`.

### REQ-10. Фильтрация
Таблица поддерживает фильтрацию по:
- Руководителю
- Типу дня
- Отделу (Backend / Frontend / Flutter / DevOps / QA / Technical Support / Analysis / Design / Management / HR / Marketing)

### REQ-11. Навигация по периодам
- При открытии файла автоматически отображается текущая неделя и последующие даты.
- Прошедшие периоды доступны при прокрутке влево.

### REQ-12. Масштабируемость
- Таблица корректно работает при добавлении новых записей и новых сотрудников.

## 4. Нефункциональные требования
### NFR-01. Удобство использования
- Пользователь может быстро фильтровать данные по руководителю, типу дня и отделу.
- Визуальное различие категорий однозначно и легко читаемо.

### NFR-02. Актуальность данных
- Текущая дата определяется автоматически при каждом открытии файла.

## 5. Критерии приёмки
### AC-01. Единое представление
- Все категории отсутствий отображаются на одном листе.
- Нет необходимости переходить между листами для просмотра данных.

### AC-02. Корректная цветовая индикация
- Дни отпуска окрашиваются в зелёный цвет.
- Праздничные дни окрашиваются в синий цвет.
- Больничные дни окрашиваются в красный цвет.

### AC-03. Корректный расчёт рабочих дней
- При вводе периода учитываются только рабочие дни.
- Суббота и воскресенье автоматически исключаются из расчёта.

### AC-04. Корректный расчёт отпусков
- Каждому сотруднику доступно 20 отпускных дней в год.
- Неиспользованные дни переносятся на следующий год.
- Перенесённые дни становятся недоступны после 30 июня следующего года.

### AC-05. Корректный расчёт праздничных дней
- Каждому сотруднику доступно 17 праздничных дней в год.
- Неиспользованные праздничные дни не переносятся на следующий год.

### AC-06. Учёт больничных
- Больничные дни добавляются без ограничений по количеству.
- Таблица корректно отображает факт использования больничных дней.

### AC-07. Отображение остатков
- Отображается остаток отпускных дней на текущую дату и на календарный год.
- Отображается остаток праздничных дней на текущую дату и на календарный год.

### AC-08. Валидация типа дня
- В столбце «Тип дня» доступны только значения `VACATION`, `HOLIDAY`, `SICK`.
- Ввод произвольного значения невозможен.

### AC-09. Фильтрация
- Данные можно отфильтровать по руководителю, типу дня и отделу.

### AC-10. Навигация по дате
- При открытии файла отображается текущая неделя и будущие периоды.
- Прошлые периоды доступны при прокрутке влево.

### AC-11. Стабильность при добавлении данных
- Расчёты остаются корректными при добавлении новых записей и сотрудников.

## 6. Ограничения и допущения
- Реализация выполняется в Google Sheets.
- Даты праздничных дней задаются вручную.
- Государственные и региональные праздники не определяются автоматически — вносятся вручную при необходимости.
